Music Networking App

Problem:

In can be difficult to network with other artists without connections, as well as finding likeminded people in a big city.

Proposed Solution:

A mobile app to express your talents and goals, find other artists to meet, inspire, and collaborate.

Platform

  • Android OS
  • XML (frontend)
  • Java (backend)

Role

  • Developer
  • Design

Occaison

  • Mobile Application Development
  • WWU

Repository

Implementation Goals

Advanced Matching

Query users based on preferred genre and musical identity

Instant Chat Feature

Allow users to meet and chat via Google Firebase client/server architecture

Unique Artist Profiles

Identify as a group or artist, and design a unique profile to express your interests

User Flow Process

Model

The project was revolved around the Model View Controller design pattern. Models include “Artist” and “Group” profiles, containing strings for display names, and arrays for genres, skills, and preferences.

View

The view relies on responsive frontend elements, implemented in XML with Android Studio. We ensured responsive design with constraint layouts on every screen.

Controller

Controller elements include the database manager. Since our database (Google Firebase) behaves asynchronously, we utilized database and connection controllers for function callbacks.

Wireframes

Final Implementation

Users sign up or log in

Create individual user profile, instance of “Artist” or “Band” initialized in the Google Firebase backend. Instance retrieved on login

Find other users populated in database. Query user results based on user preferred traits (genre or musical identity)
